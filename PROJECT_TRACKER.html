Tech Debt & Code Quality

  • 1

    No charting library — all charts are custom SVG/CSS

    Works for current needs but will become unmaintainable as analytics grow. Consider migrating to Recharts or Chart.js for interactive tooltips, zoom, and complex visualizations.

  • 2

    Currency symbol hardcoded as ₹ across all pages

    User can set currency code in settings but every page renders ₹. Need a formatCurrency() utility that reads from user context.

  • 3

    Spending trend uses simulated variations

    Analytics 6-month trend adds random ±15% to current monthly spend. Should use actual billing history aggregated by month.

  • 4

    No persistent job queue for reminders

    Cron uses basic node-cron. Needs external scheduler (Vercel Cron, Inngest, or BullMQ) for reliable delivery at scale.

  • 5

    Hydration warnings on auth pages (dev-only)

    React useId() mismatch between SSR and client for form components. Mitigated with suppressHydrationWarning but root cause is async Server Component boundary.

  • 6

    No API response validation on client

    Hono RPC provides type inference but runtime response validation is missing. Malformed API responses could crash the UI.

  • 7

    No error boundaries

    A single component error crashes the entire page. Need React Error Boundary components per section to contain failures gracefully.

  • 8

    No automated tests

    Zero unit tests, integration tests, or E2E tests. Critical for a financial app. Need at minimum: auth flow tests, payment tracking tests, budget calculation tests.
